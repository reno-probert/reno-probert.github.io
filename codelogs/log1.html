Code Log 1: HTML Semantic Tags

Differences Between Non-Semantic HTML and Semantic HTML

The central difference between non-semantic HTML and semantic HTML is in ease of navigation and searchability on the code itself. While both forms of HTML serve the same purpose of getting information onto a webpage and providing structure, semantic HTML allows a coder to use tags that are more descriptive of what is actually included in the content of the webpage. Semantic HTML also allows for accessibility tools to better describe the content of a webpage and provides search engine optimization based on the tags.

Examples of Non-Semantic Tags

  • <div>
  • <h1>
  • <b>
  • <i>
  • <img>

Examples of Semantic Tags

  • <article>
  • <header>
  • <strong>
  • <em>
  • <figure>

Semantic HTML follows pretty much the same general structure as standard HTML but allows for better descriptions of the contents. For example if you wanted to add an image using semantic tags. Instead of something more abstract like <img> to contain your image link, you would use <figure>and rather than adding a non descriptive <p> tag underneath to describe the image, you could use <figcaption>which clearly notes that the text is related to the image. In addition to the benefits of searchability, ease of navigation, and accessibility, semantic HTML does not have drawbacks as it works the same as regular HTML while providing the added benefits. Especially for people new to coding, semantic html can simplify coding and allow a coder to use better understandable terms to accomplish their goals.
